# `sincos` — Quarter-wave sin/cos LUT (Q15), 16-bit angle in

Quarter-wave sin/cos lookup: a 16-bit electrical angle `theta` (0..65535 =
0..2π) in, Q15 signed `sin` and `cos` out. A single `2^BITS`-entry table holds
`sin` over `[0, pi/2)`; the top two angle bits fold the quadrant (`quad[0]`
picks ascending vs mirrored index `~idx`, `quad[1]` sets the sign), and `cos`
reuses the same table at `theta + pi/2` (`+0x4000`). Provides the rotation
basis for `park` and `inv_park` in the FOC datapath. The table data
(`rtl/gen/sincos_init.vh`) is generated by `gen_rtl_params.py` from the same
rounding rule as `foc_reference.sincos_table`, so RTL and reference agree
bit-for-bit.

## Interface

| Port | Dir | Width | Signed | Reset | Semantics |
| --- | --- | --- | --- | --- | --- |
| `theta` | in | 16 | no | — | electrical angle, 0..65535 = 0..2π |
| `sin_out` | out | 16 | yes | — | `sin(theta)`, Q15 signed |
| `cos_out` | out | 16 | yes | — | `cos(theta)`, Q15 signed (= `sin(theta + pi/2)`) |

## Clocking & reset

- **Clock domains:** none — combinational lookup (no `clk`); the table is loaded
  once at elaboration via an `initial` `$include`.
- **Reset:** none — stateless (ROM contents are constant).
- **Latency:** combinational (zero cycles).

## Parameters

| Parameter | Default | Meaning |
| --- | --- | --- |
| `BITS` | `8` | quarter-wave table size = `2^BITS` entries (angle resolution). The table data `rtl/gen/sincos_init.vh` is generated for this `BITS`, so the two must change together |

## Formal contract

- **Sim-only** (`formal/sim_only.toml`): quarter-wave LUT; bit-exact vs the reference table (test_foc_math)
- Covered by the simulation tiers, not a machine proof.
- **Assumptions:** combinational (no clock/reset); `theta` spans the full 16-bit
  range; `rtl/gen/sincos_init.vh` was regenerated for the current `BITS`.

## Synthesis fit

- **Device:** ECP5. Combinational: a `2^BITS`×15-bit ROM (default 256×15) plus
  quadrant-fold/sign logic and one `+0x4000` adder for the cos tap. At `BITS=8`
  the table is small enough to map to distributed LUT-ROM (or a single EBR
  block); the fold logic is a few LUTs. No registers.

## Reuse notes

- **Language:** Verilog-2005; accepted by Verilator/yosys/Vivado/Quartus.
- **Dependencies:** none (leaf), but requires the generated include
  `rtl/gen/sincos_init.vh` (from `gen_rtl_params.py`) on the search path.
- **Pull it:** `fusesoc run motorloop:ip:sincos` (core at repo root).
