# Changelog

Semantic versioning (https://semver.org). Until 1.0.0, minor versions may carry
breaking interface changes; the per-module contracts (`rtl/contracts/`) define
the stable surface a release promises.

Releases are cut by tagging (`git tag v<x.y.z>`) and attaching the status matrix,
`formal/proof_report.md`, and the ECP5 bitstream as release assets.

## [Unreleased]

### Added — trusted-library foundation (stages 1–3)
- **Parameterized leaf IP.** Every reusable module takes its motorloop-specific
  constants as width-sized `parameter`s with sane defaults; no leaf includes the
  global `rtl_params.vh` anymore — `controller_top` is the only consumer and
  threads the generated config down. A module is now pull-in-able standalone.
- **FuseSoC cores** (`cores/`, generated by `cores/gen_cores.py`): 25 leaf IP
  cores + the top `motorloop.core` system core. `fusesoc run motorloop:ip:<name>`
  elaborates a single module with no motorloop includes.
- **Per-module contracts** (`rtl/contracts/`) on the uniform
  `notes/module-contract-template.md` shape; status matrix
  (`notes/status-matrix.md` + the generated live view) and verification plan
  (`notes/verification-plan.md`).
- **Pinned toolchain** (`toolchain.lock`) + a `ci` workflow (lint + sim + synth)
  alongside the existing `formal` workflow — all four gates from one pinned OSS
  CAD Suite tarball.
- **Formal:** two clamp proofs (`current_pi`, `speed_iq_pi`) upgraded to
  `param_scope: envelope` (parameter-generic — proven for any clamp value).
- SPDX (MIT) headers across the tree.

### Unchanged (guaranteed)
- The 401-test sim suite, the 12-PROVEN + 1-DOCUMENTED formal suite, and the
  ECP5 synth fit are byte-identical: every parameter defaults to its prior value.

### Known gaps (tracked for stages 4–5)
- **Timing:** the FOC datapath is unpipelined — Fmax ≈ 3.3 MHz < 25 MHz. A real
  high-speed board build needs `foc_core`/`circle_limit` pipelined.
- **Validation:** verification only; no silicon-correlation tier yet.
