# Robotics IP Checklist — shippable HDL blocks

The product is **robotics HDL blocks that ship with claims, assumptions, proofs,
tests, timing docs, and integration wrappers.** The ten layers below make those
blocks easy to *consume* (FuseSoC, Bender), *trust* (Verible, cocotb,
SymbiYosys, OpenLane), *integrate* (AXI, Wishbone), and *review* (SPDX/REUSE,
IP-XACT). Companion to [trusted-library-checklist](trusted-library-checklist.md)
(which delivered the parameterized leaf IP these build on).

**The north star (do not skip this).** *Do not let the standards become the
product.* Every item here earns its place only by making a **named, real block**
easier to consume/trust/integrate/review. A standard added with zero consuming
block is scope creep, not progress. The invariant that must never regress is the
per-block bundle:

> **claim (contract) · assumptions · proof-or-declared-sim-only · tests · timing
> doc · bus wrapper**

## Already shipped (from the trusted-library refactor)

- **#1 FuseSoC cores** — 25 leaf cores + the top system core exist (`cores/`,
  `motorloop.core`); standalone elaboration verified. *Refinement remains* (per-core
  targets, layout) — see Stage 1.
- **#2 manifest as truth source** — `formal/manifest.toml` + the generated
  `formal/proof_report.md` (12 PROVEN + 1 DOCUMENTED). *Elevate to THE source* —
  Stage 2.
- **#3 lint** — Verilator `--lint-only` is a CI gate; **Verible not yet** — Stage 3.
- **#6 SPDX/REUSE** — the tree is **REUSE-3.3 compliant** (`reuse lint` green,
  `LICENSES/`, `REUSE.toml`). *Add the CI gate* — Stage 6.

## Two prerequisites these stages surface (promote, don't defer)

- **Pipeline `foc_core`** (the library plan's stage 4). The FOC datapath caps at
  **Fmax ≈ 3.3 MHz** (`synth/synth_report.md`). This is now a *hard dependency*
  for a credible AXI-Stream sample rate (Stage 5) and any ASIC timing (Stage 10),
  not a someday-item.
- **A silicon-correlation tier** (stage 5 of the library plan) is orthogonal but
  is the trust ceiling; OpenLane "smoke" (Stage 10) is *not* a substitute for it.

## Definition of done (the product, not the standards)

Every block tagged "reusable" ships the full bundle and is: consumable by
`fusesoc run` **and** `bender`; lint-clean (Verilator + Verible); proven **or**
explicitly sim-only in the manifest; cocotb-tested where it has a protocol/FSM;
wrapped on AXI-Lite/AXI-Stream/Wishbone where an integrator needs it; documented
with a WaveDrom timing diagram; exportable to IP-XACT; and ASIC-synthesis-clean
under OpenLane. No layer is "done" until it points at a real block it serves.

## Design decisions (pre-resolved)

- **The block-value gate.** Each stage names the block(s) it serves and shows the
  before/after consume/trust/integrate/review win. PRs that add a standard with
  no consuming block are rejected.
- **One source per fact.** The manifest owns proof facts; the AXI-Lite register
  map owns the register layout (IP-XACT + docs + the regfile derive from it); the
  cores own the file lists (Bender + IP-XACT mirror them). No fact authored twice.
- **Honest labels.** "OpenLane smoke" = synthesizable + clean SDC + no latches, at
  a stated clock — *not* a tapeout or validation. Wrappers are proven for protocol
  legality, not performance.

## Stage 6.5 — Pipeline `foc_core` (the surfaced prerequisite) ✅
*Trust + integrate.* Serves: `foc_core`, the AXI-Stream rate, the OpenLane smoke.
- [x] Pipelined the Clarke→Park→PI→circle-limit→inv-Park→SVPWM chain into a

## Findings

All ten stages (+ 6.5) landed. The product is still **robotics HDL blocks that
ship with the bundle** — claim · assumptions · proof-or-sim-only · tests · timing
· wrapper; the standards below each earned their place against a named block.

**The keystone — pipelined `foc_core` (Stage 6.5).** The synth flow surfaced the
real bottleneck: the whole Clarke→…→SVPWM chain was combinational, and
`circle_limit`'s 16-iteration isqrt + two 32-bit divisions capped the system at
**Fmax ≈ 3.3 MHz**. Walking the chain over a registered sequencer and replacing
the combinational limiter with the **sequential, bit-exact `circle_limit_seq`**
(reusing the existing `divider32`) took the system to **41.27 MHz** (clears the
25 MHz target) *and dropped LUT4 from 13791 to 3914* — the huge combinational
divide/sqrt collapsed into a small reused datapath. Behaviour is preserved
exactly: the PI integrators still advance once per `update` with identical error
and freeze(sat), so all **400 tests stayed green** (latency-aware) and the duties
just appear a fixed, sub-sample-period latency later. This was the hard
dependency for a credible AXI-Stream rate (Stage 5) and the ASIC smoke (Stage 10).

**Which layer made which block easier.**
- *Consume* — FuseSoC gives each of 29 leaves a standalone `fusesoc run`; Bender
  mirrors the same source map for PULP/RISC-V SoCs.
- *Trust* — the coverage gate (Stage 2) makes "unproven" impossible to ship
  silently; Verible keeps every block review-clean; cocotb exercises the
  protocol/FSM/datapath blocks in isolation (9 green), and `circle_limit_seq`'s
  cocotb **equivalence** test is the correctness claim that justifies the whole
  pipelining.
- *Integrate* — the AXI-Lite / AXI-Stream / Wishbone wrappers are each **formally
  proven** for protocol legality (not just simulated), so an SoC integrator drops
  the controller onto a standard fabric with a proof, not a hope.
- *Review* — REUSE makes the licence machine-checkable (vendor datasheets
  correctly proprietary, not MIT); IP-XACT (validated against the 1685-2014
  schema) lets an EDA tool ingest the register map from the one source it already
  owns.

**Where a standard threatened to become the product (and what we did).**
- *Per-core FuseSoC `sim`/`formal` targets (Stage 1).* Adding them would have
  duplicated the working cocotb runner and the formal manifest. We shipped the
  per-core **`lint`** target (a real consume win) and kept sim/formal at the
  suite level — one source each, no duplication.
- *Formal-proving `circle_limit_seq` (Stage 6.5).* The combinational
  `circle_limit` is already **DOCUMENTED-not-proven** (the isqrt+divide is
  intractable for the open SMT engines); an end-to-end miter through the same
  math over 50+ cycles would be worse. We made the **cocotb equivalence test the
  verification** (bit-exact vs the reference over ~1000 cases incl. the
  saturation boundary, plus a 61k-pair sweep) and declared it sim-only with that
  precise reason — honest, and consistent with the existing precedent.
- *Verilator `-Wall` as a gate (Stage 3).* The intentional wide-intermediate
  truncations make `-Wall` noisy; rather than contort the proven leaf RTL we kept
  Verilator **advisory** and made **Verible** the enforced style gate.
- *OpenLane (Stage 10).* No Docker/OpenROAD/PDK here, so we shipped a real
  **yosys ASIC-readiness smoke** (14/14 blocks: maps to gates, 0 latches, no
  multidriver/loops, gate counts) that gates CI, and **wired** the full sky130
  flow for where it's installed — framed honestly as *synthesizable + clean*, not
  a tapeout. The credible speed number stays the FPGA flow's real 41.3 MHz.

**One scar worth keeping.** `scripts/gen_docs.py` originally `rmtree`'d its output
dir set to `docs/`, which deleted the (gitignored, non-redistributable) vendor
datasheets. Fixed: the generator writes `site-src/` with an `assert` that refuses
any other dir; `docs/` is never the mkdocs source. Lesson: a generator that
deletes its target must never aim at a hand-curated tree.
